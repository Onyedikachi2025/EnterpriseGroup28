Dataset Overview

We obtained our dataset to use for our analysis from Kaggle. It features structured information about the music available on Spotify including both metadata from Spotify and audio features of the songs themselves. The dataset is designed to aid analysis of music trends and explore common audio features. The overall dataset includes 4833 rows and 30 columns of data. Each row represents a single track while each column represents a feature of the track that can be measured in some way. These variables include a mix of categorical and numerical data including variables such as the track name or ID, the artists' names, the albums off which the tracks are from, their popularity rating, the tempo and duration, and many more different variables. Most of the core variables are generated by Spotify's own audio analysis algorithms and are useful for analysis and machine learning as they are already standardised and scaled in most cases.

Data Cleaning Summary

A00020993 — Terry Onwubuilo

When performing the data cleaning, I used one excel sheet which is the popular music charts data sheet which contains attributes such as liveness, danceability and track popularity. These will be the few ones I will be using for data cleaning. First I will use the Trim command to get rid of any data that is missing. Then I will get the average, min, max and SD for all these attributes. I got rid of URL, Link etc. as it was unnecessary and kept danceability, playlist genre, liveness, track artist, track popularity, duration Ms and track name. I used MAX(), MIN() and SDEV.P() to find the max, min and standard deviation of the dataset.
